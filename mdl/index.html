MDL Authoring Notes

Palette Slots Are Data

Quake skins store palette indices, not best-fit colors. If a palette slot changes, every pixel using that slot changes with it. Remap the skin pixels when changing palette order.

Fullbright And Alpha

Indices 224-254 glow under fullbright rendering. Index 255 is normally reserved for cutout transparency and needs the MDL transparency flag for embedded skins in QSS-M. Treat accidental 224-254 pixels as contamination unless they are intentional glow.

Skin Size And Sampling

Classic MDL tooling is fragile above 480 pixels of skin height, while very wide skins tend to be safer. MDL and MD3 UVs should land on texel centers, and nearest/closest filtering avoids edge bleed while checking pixel art.

Debug One-Texel Seams

A thin black line is often a UV alignment error rather than missing geometry. The bundled ID1 Grunt (progs/soldier.mdl) has a known line along its left side because the back-facing islands on the atlas's right half sit one texel too far right; moving those islands one texel left aligns them. The UV Texel Nudge tool can apply that exact back-face S offset and reports shared vertices that also affect the front. Treat this as a model-specific correction, then check other seams with nearest filtering before moving UVs.

Flood Fill Pixel

QuakeSpasm-family engines still flood-fill alias skins from texel 0. Keep that texel out of the sampled artwork or use a guard texel so flood fill cannot rewrite visible art.

UVs Can Raise Vertex Counts

Texture seams and split UVs can make the exported MDL carry more vertices than the plain mesh suggests. A model that loads untextured can exceed limits once UV mapping is included.

Packed Vertex Precision

Vanilla IDPO MDL stores mesh topology once, then each pose stores an unsigned-byte X, Y, and Z plus a normal index for every vertex. Header scale and origin map each axis onto only 256 coordinate levels (0-255), so a large all-pose bounding box produces coarse steps. This format quantization can cause snapping or “vertex wobble” even with interpolation disabled; lerp only exposes motion between separately rounded poses. Keep bounds tight, avoid distant outlier or hide-away poses, and prefer MD3 when a large or precise animated mesh needs more positional resolution.

Blender Export Habits

Use Blender Unit System None when treating 1 Blender unit as 1 Quake map unit. Material slots can become MDL skin slots, but each skin must match the same mesh and UV layout. MDL export is happiest with one combined mesh.

Frame Numbers And Names

MDL poses and QuakeC frame indices are zero-based, while many Blender MDL workflows place pose 0 on timeline frame 1. Frame ranges are inclusive—77 through 97 contains 21 poses and normally exports as MDL indices 0-20. Many exporters sample Blender's configured playback/export range, and some also expect the playhead at its first frame, so extend the end when appending poses and rewind before export. Keep an explicit range map and verify the exported count after every insert or deletion. QuakeC $frame names are readable compile-time aliases for the same numeric indices used by self.frame and self.weaponframe, not model precaching.

Baked Shape-Key Editing

MDL animation has no bones. A relative Basis edit adds the same object-space vertex offset to every pose, so lengthening a sword or head there will not follow its changing local rotation and can warp later poses. Keep Relative mode and active-key editing enabled; for a local correction, the Blender timeline frame and selected pose key must match, with that key at 1 and the other pose keys at 0. Otherwise an edit can appear to do nothing or alter the wrong pose. Deleting pose keys may leave keyed influence curves behind, so inspect the Action/F-curves and rebuild the 0→1→0 weights when retiming. See the Blender shape-key walkthrough.

Adding Poses And Attachments

To append a pose, add a relative shape key, key its value to 0 before the intended frame, 1 on that frame, and 0 immediately after it, then extend the export range and verify the result in the frame tree. In this tool, Duplicate Pose inserts an editable copy beside the selected pose, while Duplicate Sequence appends a separately named copy of the active sequence. Joining new un-keyed geometry to a baked model does not invent motion for it. In Blender, a separate rigid helper can follow translation and rotation by vertex-parenting it to three non-collinear animated vertices, but do not assume an MDL exporter will bake a separate object correctly; use an armature and rebake, or an MD3 workflow, for major or multi-object changes.

Sequences Versus Frame Groups

Importers and viewers may collect numbered names such as stand1, stand2, and stand3 into a display sequence. That organization is not an MDL frame group. A true group is one top-level frame containing multiple timed poses that cycle automatically, as used by flames; self.frame selects the top-level group index rather than each internal pose. Blender add-ons can flatten, duplicate, freeze, or merge these groups, so recheck group boundaries and pose counts after export.

Verify The Loaded Asset

An apparently failed export may be the wrong file loading. Within one QSS-M game directory, package content has priority over regular loose files, so a model inside a PAK can mask the copy in progs/. Confirm the active game and model path, then isolate the conflicting package or test from a later, higher-priority game directory before debugging the animation.

Rigs And Variant Models

For Quake-style remakes, a Remaster MD5 armature and animations can be a useful base: import the MD5 mesh, apply its animations to that armature, fit the new mesh to a neutral pose, and expect manual weight and pose cleanup. Mirror tools help when reconstructing a symmetrical neutral pose from an old MDL. Object animation is quick for blockouts, but an armature is easier to revise and rebake. Viewmodels can omit hands or use low-detail fists as part of the weapon; hand and elbow IK targets simplify placement, while extending an arm beyond the elbow improves melee range at the cost of more cleanup and more visible MDL wobble.

Append Or Swap Variants

A death or damage variant can be appended as more poses in the original MDL, preserving its model path and existing sequences, or placed in a separate MDL selected with QuakeC setmodel. After duplicating a sequence, Collapse Vertex Region can select its head vertices from one reference pose and collapse those same indices to the animated boundary in every copied pose without welding them. Verify the intended degenerate triangles remain. A separate headless model can instead delete the head and close the neck, costs a model swap and new code sequence, and is usually easier to rig, texture, and maintain.

Preview Engine Interpolation

Quake engines interpolate each corresponding vertex in a straight line between stored poses; Blender's stepped shape-key preview and its Bézier or easing controls do not reproduce that runtime path. Large rotations, translations, or rescaling can compress, invert, or apparently remove volume between frames. Add intermediate poses and use smaller angular steps. For alternating Nailgun flashes, two meshes that expand and collapse in place avoid dragging one flash across both barrels. Collapse/move/expand tricks can also create teleports or card swaps, but keep topology fixed and remember that distant poses worsen MDL quantization.

Animation Timing

Original QuakeC commonly advances alias animation at 0.1-second intervals (10 fps). A 0.05-second step (20 fps) is a practical smoother starting point for fast weapon motion; 30-40 fps can reduce rotation distortion further but costs more authored poses. Timing can vary by QuakeC frame range, so the main action may remain at 10 fps while a muzzle flash uses a shorter nextthink interval. Test the actual engine because interpolation and network timing affect the result.

Forcing Jump Cuts

Swapping an entity between two separate, identical MDL paths resets animation interpolation in QSS-M because it does not lerp across a model change. This can force stop-motion poses, instant repositioning, spinning-barrel stops, and concealed reload resets without relying on a player's cvars; test other engines before depending on it. In QSS-M, r_lerpmodels 0 disables model interpolation, the default value 1 honors muzzle-flash and no-lerp resets, and 2 keeps interpolation through those exceptions.

Viewmodel Framing

A useful Blender starting camera is at world origin, rotated X 90°, Y 0°, Z 270°, looking toward the positive X export axis. Blender camera FOV is not numerically interchangeable with Quake's FOV: aspect ratio, sensor fit, engine FOV adaptation, status-bar mode, weapon scale, and bobbing all affect the result. Match an in-game screenshot, then test low and high FOV in-engine—especially hidden muzzle flashes, backfaces, forearms, and off-screen moving parts.

MD3 Replacement Gotchas

MD3 replacements still need a matching MDL name in QuakeSpasm-family engines. Keep the placeholder MDL flags intentional, use lowercase texture names for Linux-safe skin files, map MD3 object names to texture paths, and use black base pixels under glow overlays when the glow color should stay exact.

MD3 Fallback Planning

A null MDL is fine as a placeholder, but players without enhanced MD3 replacements enabled will see nothing. A visible message fallback MDL is safer for MD3-only assets. If MDL and MD3 both ship, use the same rig where possible but plan for MDL's single atlas and combined-mesh limits.

MD3 Transparency Caveats

Weapon-viewmodel MD3 alpha is usually the safe case. Non-viewmodel MD3 alpha can be engine-version sensitive, especially with overlapping double-sided planes, alternate skins, or placeholder MDL flags. Test foliage, glass, and cards in-engine from several angles.

MD3 Export Traps

Do not make the first MD3 frame collapsed or scaled to zero; use a tiny visible scale or a reference pose instead. A triangulate step can prevent surprise sliced angles, and very large MD3 objects still need engine-side size testing even when vertex counts are high.

Reserved Palette Indices
In Quake's 256-color palette, the main reserved entries are the fullbright range 224-255, which stay bright regardless of lighting, the transparency index 255, which is commonly used as the masked "hole" color for sprites, fence textures, and some model rendering, and the two player colormap translation bands 16-31 for shirt/top colors and 96-111 for pants/bottom colors.
There is also one special case for console characters, where both palette index 0 and 255 are treated as transparent. In practice, that means the usable visible fullbright colors are really 224-254, because 255 is usually consumed by transparency rather than shown as a normal glowing color.
